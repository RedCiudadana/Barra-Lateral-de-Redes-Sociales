Install

All of the files that you need to work with are in the Main folder. In order to install the sidebar on your html page, you will need to do the following steps:

1. Copy the sidebar assets folder, "super-sidebar", and add it next to your html file.

2. Copy the sidebar html code and add it to your html file:

Select a template and open it in a text editor.

a. Copy the assets code and add it at the end of the <head> tag on your html page.

b. Copy the markup code and add it at the end of the <body> tag on your html page.

c. Copy the build code and add it at the end of the <body> tag on your html page.

Detailed instructions

1. Copy the sidebar assets folder, "super-sidebar", and add it next to your html file.

This folder contains the files needed by the sidebar such as the CSS, JavaScript and icon files. The folder needs to be with your html file so that the sidebar can load the necessary assets.

2. Copy the sidebar html code and add it to your html file:

There are multiple templates that the sidebar comes with, which provide a starting point in customizing it. Once you have decided which template to use, open the html file in a text editor (such as Sublime Text).

After you open the file, you will see that the needed sidebar code is shown by comments:

a. Copy the assets code and add it at the end of the <head> tag on your html page.

<!-- Start of sidebar assets code -->
<link href="super-sidebar/css/super-sidebar.min.css" type="text/css" rel="stylesheet">
<link href="super-sidebar/font-awesome/css/all.min.css" type="text/css" rel="stylesheet">
<script src="super-sidebar/js/jquery-3.4.1.min.js" type="text/javascript"></script>
<script src="super-sidebar/js/velocity.min.js" type="text/javascript"></script>
<!-- <link href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/5.8.2/css/all.min.css"... -->
<!-- <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.4.1/jquery.min.js"... -->
<!-- <script src="https://cdnjs.cloudflare.com/ajax/libs/velocity/1.5.0/velocity.min.js"... -->
<script src="super-sidebar/js/jquery.super-sidebar.min.js" type="text/javascript"></script>
<!-- End of sidebar assets code -->

This code specifies the necessary files to be loaded by the page.

If you already have Font Awesome, jQuery or Velocity on the page, do not add them again. In this case, make sure they are added on the page before the sidebar JS file.

Commented out are alternative ways of loading Font Awesome, jQuery and Velocity, from CDNs (Content Delivery Networks). The reason why you would want this is because the loading times would be faster for the user if he were to load the files from a CDN server rather than from your own site. Or, if the user visited sites using the same practice, he might already have the files already cached in his browser. I have added these as a useful option for people who want them, but I have not enabled them by default because it will always require an Internet connection to preview the sidebar. But, when deploying it on the web, it is a good idea to use these faster loading options instead.

b. Copy the markup code and add it at the end of the <body> tag on your html page.

<!-- Start of sidebar markup code -->
<div class="super-sidebar">
    <ul class="sb-bar">
        <li class="sb-b-facebook">
            <a data-share="facebook">
                <div class="sb-icon fab fa-facebook-f"></div>
                <div class="sb-label">Share on Facebook</div>
            </a>
        </li>
    ...
</div>
<!-- End of sidebar markup code -->

This is the bulk of the sidebar html code and it specifies all of the content of the sidebar, such as buttons, labels and windows content.

After you are done with the installation, you should look over the Buttons section to learn how to customize this code.

c. Copy the build code and add it at the end of the <body> tag on your html page.

<!-- Start of sidebar build code -->
<script type="text/javascript">
    jQuery(document).ready(function() {

        jQuery(".super-sidebar").superSidebar({
            // Main
            position: ["left", "center"],
            offset: [0, 0],
            buttonShape: "square",
            ...
        });

    });
</script>
<!-- End of sidebar build code -->

This is the JavaScript code that builds the sidebar functionality by calling the plugin and passing the settings to it.

To see all of the available settings, their meanings and possible values, see the Settings table.

Now that you have the sidebar up and running on your page, you can start to customize it as you like.

Customize

When customizing the sidebar, you can change the HTML markup code to customize the content (buttons, subbars, windows), you can change the settings to customize the functionality, you can edit the CSS file to change anything about the look that you want and, for more advanced users, you can also edit the sidebar JavaScript file.

Buttons

A button of the sidebar can be of multiple types:

  • Share: Shares the page on a service, such as a social network.
  • Link: Links to a page, such as the profile page on a site.
  • Special: Has a special function on click.
  • Window: Opens an overlay window.
  • Subbar: Opens a sublist of buttons.

The markup code for a button is an <li> element inside <ul class="sb-bar"> or the <ul> of a subbar button:

<ul class="sb-bar">
    <li class="sb-b-facebook">
        <a data-share="facebook">
            <div class="sb-icon fab fa-facebook-f"></div>
            <div class="sb-label">Share on Facebook</div>
        </a>
    </li>
    ...
</ul>

Button color

The color of the button (and the label) is controlled by the sb-b-facebook class of the <li> element:

<li class="sb-b-facebook">
    ...
</li>

If the color for the button that you want is not available by default, you can add it yourself in the CSS code:

.sb-b-facebook .sb-icon,
.sb-b-facebook a:hover .sb-icon,
.sb-b-facebook .sb-label {
    color: #3b5998;
    background-color: #3b5998;
}

You can use a site such as brandcolors.net to find the color code for the service that you want.

Font Awesome icons

The button icon is controlled by the fa-facebook class of the sb-icon element:

<div class="sb-icon fab fa-facebook-f"></div>

The sidebar uses Font Awesome which has available over 1500 free icons that can be used.

You can view all of the available icons here: https://fontawesome.com/icons?d=gallery&m=free.

Custom image icons

Additionally, the buttons support using custom images as icons as well:

<div class="sb-icon">
    <img src="super-sidebar/images/icons/facebook.png" alt="Facebook icon">
</div>

Share button

A share button shares the current page on a service, such as a social network.

The following is the markup code of a share button:

<li class="sb-b-facebook">
    <a data-share="facebook">
        <div class="sb-icon fab fa-facebook-f"></div>
        <div class="sb-label">Share on Facebook</div>
    </a>
</li>
  • The sb-b-facebook class specifies the color of the button and the label. This is defined in the CSS file.
  • The data-share attribute specifies the service on which the page should be shared.
  • The fab fa-facebook-f classes specify the button icon from the Font Awesome library.
  • The sb-label element contains the button label that is displayed on mouseover. The label is optional and the element can be removed if not necessary.

Add a share service

Apart from the default share services, you can add new ones through the JS settings:

shareServices: {
    "facebook": {
        type: "popup",
        url: "https://www.facebook.com/sharer.php?u={URL}",
        window: {width: 700, height: 450, top: 200, left: "{CENTER}"}
    }
}
  • The "facebook" string is the name of the service. This will be used in the data-share attribute of the buttons.
  • The type property specifies the type of the share functionality:
    • "popup": Open in a popup window.
    • "blank": Open in a new window.
  • The url property is the share URL specific to the service that you want to add.
    This can contain 2 special strings, {URL} and {TITLE}, that the sidebar will replace with the page URL and the page title respectively.
  • The window propery contains the popup window parameters.
    • For top and left, the special string {CENTER} can be used to center the popup vertically and horizontally in relation to the screen.
    • Applicable only for type: "popup".

Link button

A link button simply links to a page, such as the profile page on a site.

The following is the markup code of a link button:

<li class="sb-b-facebook">

    </a>
</li>
  • The sb-b-facebook class specifies the color of the button and the label. This is defined in the CSS file.

  • The fab fa-facebook-f classes specify the button icon from the Font Awesome library.
  • The sb-label element contains the button label that is displayed on mouseover. The label is optional and the element can be removed if not necessary.

Window button

A window button links to an overlay window that opens over the page.

The following is the markup code of a window button:

<li class="sb-b-contact">
    <a href="#super-sidebar-contact" target="_self">
        <div class="sb-icon fas fa-envelope"></div>
        <div class="sb-label">Contact form</div>
    </a>
</li>
  • The sb-b-contact class specifies the button color. This is defined in the CSS file.
  • The #super-sidebar-contact is the window id and specifies that this button links to a window as opposed to another page.
  • The fas fa-envelope classes specify the button icon from the Font Awesome library.
  • The sb-label element contains the button label that is displayed on mouseover. The label is optional and the element can be removed if not necessary.

Window markup code

The markup code of a window goes in the <div class="sb-overlay"> element.

If there are no windows used, this element can be removed.

The following is the markup code of a window:

<div id="super-sidebar-contact" class="sb-window">
    <div class="sb-head">
        <div class="sb-title">CONTACT</div>
        <div class="sb-close"></div>
    </div>
    <div class="sb-body">
        ...
    </div>
</div>
  • The super-sidebar-contact is the unique id of the window.
  • The sb-title element contains the title that appears in the window header.
  • The sb-body element contains the content of the window. This can be any html content.
  • In the Settings table you can find different window settings.
Window size

Additionally, the window element can contain any of the following classes, in order to set a width dimension:

  • sb-normal-window: 500px
  • sb-big-window: 700px
  • sb-large-window: 1000px
  • sb-large-window: 400px
<div id="super-sidebar-contact" class="sb-window sb-normal-window">

Or you can set a custom size in the CSS code using the window ID:

#super-sidebar-contact {
    max-width: 420px;
}
Window position

The position of the windows is set from the windowPosition JS setting but it is possible for each window to override the general setting and have a different position, by using the data-position attribute:

<div id="super-sidebar-contact" ... data-position="center-top">

The possible values are similar to the setting values.

The same applies for the windowOffset setting, using the data-offset attribute:

<div id="super-sidebar-contact" ... data-offset="10-20">

Window form

Among other content, a window can contain a form that will be handled by the sidebar plugin itself:

<form class="sb-form" action="super-sidebar/php/contact.php" data-id="contact">
    <label>Your name:</label>
    <input type="text" name="name" required>
    ...
    <div class="sb-submit">Send message</div>
    <div class="sb-status"></div>
</form>
  • The sb-form class specifies that this is a sidebar form and should be handled by the plugin.
  • The action attribute specifies the file that will process the form when submitted.
  • The data-id attribute specifies the unique id of the form. This can be used in the settings to pass data to it.
  • The form can contain regular HTML form elements.
  • The required attribute specifies that this field is required.
  • The sb-submit element will submit the form when clicked.
  • The sb-status element shows different status messages (error, working, success, etc).
    These messages can be customized for each form through the settings. For more information, see below.
  • The data is submitted to the processing file using the POST method.
Form data

Using the settings, it is possible to pass data to the form:

formData: {
    "contact": {
        post: {
            test: "something",
            other: "other value",
            ...
        },
        status: {
            empty: "Please fill all of the required fields.",
            badEmail: "The email format is incorrect.",
            working: "Sending message, please wait...",
            success: "The message was successfully sent!",
            error: "There was an error! The message was not sent."
        }
    }
}
  • The "contact" string is the unique id of the form. This is the one specified in the data-id attribute.
  • The post property contains other values that should be submitted to the processing file alongside the form values.
    This is optional and can be ommited if there is no other data which should be submitted.
  • The status property specifies the different status messages of the form.
    This is also optional. If there are no status messages specified for the form, the default status messages will be used instead.
Processing file response

Once the file processing the form has done its job, it should send a response back with the following JSON format:

Success response:

{
    status: true
}
  • It can also contain a message but this is optional, since a success message is already defined.

Error response:

{
    status: false,
    message: "Error message here."
}
  • The message is optional, since a general error message is already available, but it is best to send back a specific error message about what went wrong.

Settings

These are settings that you place in the JavaScript build code:

jQuery(".super-sidebar").superSidebar({
    // Main
    position: ["left", "25%"],
    offset: [0, 0],
    buttonShape: "square",
    ...
});

The table below shows all of the settings that are available, their default and possible values, as well as a description about each. It is not necessary to specify all of the settings in the build code. When a setting is not specified, it will automatically take the default value.

Name Default value Possible values
// Main
position ["left", "center"] [<horizontal>, <vertical>], ["left", "center"], [10, 20]

The position of the sidebar. The format is [<horizontal>, <vertical>]. Horizontal values are "left" and "right" and the vertical values are "top", "center" and "bottom".

Additionally, the values can be exact numbers like 10 (px) or percentages like "20%".
offset [0, 0] [<horizontal>, <vertical>], [10, 20]

The offset in position starting from the position setting. These values are added to the positions calculated previously.

The format is [<horizontal>, <vertical>] where the values are numbers.
buttonShape "round" "square", "round", "rounded", "rounded-out"

The shape of the buttons. It also determines the shape of the labels.
buttonColor "default" "default", "blue", "red", "yellow", "green", "white", "black"

The color of the buttons.

These are only a few default color options, you can edit the CSS code and add any colors you want.
buttonOverColor "default" "default", "blue", "red", "yellow", "green", "white", "black"

The color of the buttons on mouseover.

These are only a few default color options, you can edit the CSS code and add any colors you want.
iconColor "white" "default", "blue", "red", "yellow", "green", "white", "black"

The color of the button icons.

These are only a few default color options, you can edit the CSS code and add any colors you want.
iconOverColor "white" "default", "blue", "red", "yellow", "green", "white", "black"

The color of the button icons on mouseover.

These are only a few default color options, you can edit the CSS code and add any colors you want.
labelEnabled true true, false

If the button labels should be enabled.
labelColor "match" "default", "blue", "red", "yellow", "green", "white", "black", "match"

The color of the button labels.

If set to "match", it will correspond with buttonOverColor.

These are only a few default color options, you can edit the CSS code and add any colors you want.
labelTextColor "match" "default", "blue", "red", "yellow", "green", "white", "black", "match"

The color of the button labels text.

If set to "match", it will correspond with iconOverColor.

These are only a few default color options, you can edit the CSS code and add any colors you want.
labelEffect "slide-out-fade" "fade", "slide-out", "slide-out-fade", "slide-in", "slide-out-out", "slide-in-in", "none"

The appearance effect of the button label.

Certain effects (like "slide-in" or "slide-out-out") cannot work with labelConnected: true and so, if chosen, they will reset labelConnected to false.

labelAnimate "default" [<time>, <ease>], [400, "easeOutQuad"], "default"

The animation parameters of the label effect. The time is in milliseconds and for the easing, see the Easing options at the bottom of the table.

Different effects have different default values so you can just set "default" and the sidebar will automatically pick the defaults for that effect.

Additionally, you can set different parameters for the show and the hide effects, as so:

labelAnimate: {
    show: [<time>, <ease>],
    hide: [<time>, <ease>]
}
labelConnected false true, false

If the button and label should be visually connected or not. If they are connected, when the label appears, it looks like it expands from the button.
sideSpace false true, false

If there should be space on the side of the bar.
buttonSpace false true, false

If there should be space between the buttons.
labelSpace false true, false

If there should be space between the label and the button.
// Subbar
subPosition ["circular",105,-90,90] "under", "side", ["circular", <radius>, <start rad>, <end rad>]

The position of the subbar. It can be "under" the button, on the "side" of the button or "circular", around the button.

If it is "circular", there are other values that can be included such as the radius of the circle and the start and end radians.
subEffect "slide" "fade", "slide", "linear-fade", "linear-slide", [<effect>, <interval>], "none"

The appearance effect of the subbar.

For the "linear-fade" and "linear-slide" effects, the value can be an array with the effect name and the interval in ms between the animation of each button: ["linear-fade", 30].
subAnimate [400, "easeOutQuad"] [<time>, <ease>], [400, "easeOutQuad"], "default"

The animation parameters of the subbar. The time is in milliseconds and for the easing, see the Easing options at the bottom of the table.
subSpace false true, false

If there should be space between the subbar and the button.
subOpen "mouseover" "mouseover", "click"

If the subbar should be opened on mouseover or on click.
// Window
windowPosition ["center", "center"] [<horizontal>, <vertical>], ["center", "20%"], ["center", 20]

The position of the window. The format is [<horizontal>, <vertical>]. Horizontal values are "left", "center" and "right" and the vertical values are "top", "center" and "bottom".

Additionally, the values can be exact numbers like 20 (px) or percentages like "20%".
windowOffset [0, 0] [<horizontal>, <vertical>], [10, 20]

The offset in position starting from the windowPosition setting. These values are added to the positions calculated previously.

The format is [<horizontal>, <vertical>] where the values are numbers.
windowCorners "match" "square", "round", "match"

The type of the window corners.

If set to "match", it will correspond with the shape of the buttons.
windowColor "match" "default", "blue", "red", "yellow", "green", "white", "black", "match"

The color of the header of the window.

windowShadow true true, false

If the window should have a shadow or not.
windowDraggable true true, false

If the window should be draggable from the header or not.
// Other
showAfterPosition false <scroll position>, false

If the sidebar should be shown only after the page was scrolled beyond a certain point.
barAnimate [250, "easeOutQuad"] [<time>, <ease>], [250, "easeOutQuad"], "default"

The animation parameters of the bar. The time is in milliseconds and for the easing, see the Easing options at the bottom of the table.
hideUnderWidth false <page width>, false

If the sidebar should be hidden when the browser window is under a certain width.
shareTarget "default" "blank", "popup", "default"

How the share window will be opened. "blank" to open the page in a new browser tab/window, "popup" to open the page in a popup window or "default" to use the default setting for each button.

Which engine to use for animations.

When using "jQuery", it is no longer necessary to load the velocity.min.js file on the page.

A custom page URL to be used in the share window, instead of the current page URL.

If the setting is not specified or set to null, the current page URL is used.

A custom page title to be used in the share window, instead of the current page title.

If the setting is not specified or set to null, the current page title is used.

Adds new or changes the existing sharing services.

More information is explained in the Add a share service section.
formData { } <forms data>

Specifies data to be used by the window forms.

More information is explained in the Window form section.

CSS code

  • You can edit the CSS code to customize anything about the sidebar look that you want.
  • The sidebar CSS file is called super-sidebar.min.css and is located in the super-sidebar/css folder. This file is minified. You can find the source file next to it, in the same folder.
    Copy the contents of the source and replace all of the code in the minified file. Now you can make changes and see them live.
  • The code is ordered to resemble the structure of the sidebar HTML code, to make it easier to understand and work with. Comments are at the head of every section and the styles inside are about elements in that section.
  • You can play around and experiment with customizing the CSS code in different ways. Don't be afraid of breaking anything because you always have the original file as a backup, in case anything goes wrong.
  • Once you are done with the customizations and are ready to deploy the file online, it is a good idea to minify it so that it will load faster. You can use a service such as cssminifier.com.
    Remember to save a copy of the source for yourself, in case you want to make further modifications later.

JS code

  • For more advanced users, it is also possible to edit the JavaScript code in order to customize anything about the sidebar functionality that you want.
  • The sidebar JS file is called jquery.super-sidebar.min.js and is located in the super-sidebar/js folder. This file is minified. You can find the source file next to it, in the same folder.
    Copy the contents of the source and replace all of the code in the minified file. Now you can make changes and see them live.
  • The bulk of the code is in the build function, where the gallery is initialized and its functionality built.
    You can also see the default settings, sharing services as well as other data.
  • Don't be afraid to experiment with modifying the code as you will always have the original file as a backup, in case anything goes wrong.
  • After you have made the changes that you want and are ready to deploy the file online, it is a good idea to minify it so that it will load faster. You can use a service such as jscompress.com.
    Remember to save a copy of the source for yourself, in case you want to make further modifications later.

Problem

If you have a problem with the sidebar, a few common things to do are:

  • Make sure that jQuery is not added multiple times on the page.
    If you already have jQuery on your page, remove it from the sidebar assets code. But make sure that your jQuery is added before any sidebar JS file.
  • Open the browser developer tools (F12) and see if there are any errors shown in the console.
    If there are any error messages shown, try to read and understand what they say. Try to google them to find out more information about what they mean or send the messages to me and I will explain them to you.
  • Have a look over the sidebar message board, in case the problem has been discussed before and solved.
  • Make sure that you have added all of the sidebar code (assets, markup, build) to your page.
  • Make sure that all of the used files are accessible and loading well (CSS, JS).
